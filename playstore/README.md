# Play Store Release Hub

This folder is the release control center for MuslimOS.

Use it for:

- QA checklists
- store listing copy
- privacy and compliance docs
- Play Console submission drafts
- screenshot and asset requirements
- release notes and rollout logs
- helper scripts that support publishing

Do not place the native Android project here.

Why:

- Capacitor expects the Android shell at `/android`
- Android Studio workflows, signing, and CI are simpler when the native app stays at the repo root
- `/playstore` should remain process documentation, not runtime source

## Expected repository layout

- `/android`
  Native Android shell generated by Capacitor
- `/playstore`
  Release process, assets, QA, and store metadata
- `/src`
  Main app source

## First-time Android setup

1. Install Capacitor packages:
   `npm install @capacitor/core @capacitor/cli`
2. Generate the Android shell:
   `npx cap add android`
3. Build and sync the web app into Android:
   `npm run android:build`
4. Open Android Studio:
   `npm run android:open`

## Release commands

- `npm run qa:release`
  Build plus current Playwright regression suite
- `npm run android:build`
  Build web assets and sync Android
- `npm run android:apk:debug`
  Build a local debug APK for internal QA
- `npm run android:aab:release`
  Build a release Android App Bundle once signing and Java/Android Studio are set up
- `npm run android:open`
  Open the Android project in Android Studio
