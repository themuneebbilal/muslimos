# Data Safety Draft

Use this as the starting worksheet for the Play Console Data safety form.

## App account creation

- no account creation required

## Data collected

- precise location
  Only when the user grants permission for prayer times and qibla
- app activity / app info
  Not intentionally collected as a product feature
- personal info
  No name, email, phone, or account identity required by the app itself

## Data sharing

- no intentional sale of user data
- no intentional third-party advertising data sharing

## Data storage

- core user state is primarily stored on-device
- remote content may be fetched for audio, tafseer, and hadith APIs

## Security

- transport is expected over HTTPS
- sensitive journal content is currently local-device data, not cloud-synced

## Review before submission

- verify exact Play Console answers against real network behavior
- confirm whether any analytics, crash reporting, or SDK data collection has been added later
