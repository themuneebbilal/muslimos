MuslimOS Privacy Policy

MuslimOS is designed to keep core personal usage data on-device wherever possible and use permissions only for user-facing features that require them.

Last updated: March 15, 2026

Overview

MuslimOS provides prayer times, Quran reading, recitation playback, hadith, qibla direction, worship tools, Islamic calendar features, and private journaling. We aim to minimize data collection and keep personal state local to the device unless a feature specifically requires remote content delivery.

Data We May Access

  • Location: used for prayer times and qibla direction when the user grants permission.
  • Audio playback state: used for Quran recitation playback and player controls.
  • Local app data: used for bookmarks, favorite surahs, reading position, journal content, streaks, reciter choice, and settings.

How Data Is Stored

  • Bookmarks, favorites, settings, journal entries, and reading state are stored locally on the user’s device.
  • MuslimOS does not currently require account creation.
  • MuslimOS does not sell personal data.

Network Requests

Some features may request content from third-party services, including Quran recitation audio, tafseer content, and API-backed hadith data where applicable.

Permissions

  • Location permission is optional and is only used for location-sensitive features.
  • Notification permission may be requested in future versions if prayer reminders are introduced.

User Control

  • Users can deny location access.
  • Users can clear app data from device settings.
  • Users can remove saved bookmarks and favorites inside the app.
