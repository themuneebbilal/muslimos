# MuslimOS

Prayer times, full Quran reader, dhikr counter, and qibla compass.

## Design System
- **Fonts**: Amiri (Arabic/headers) + DM Sans (UI) — NEVER change these
- **Colors**: Emerald (#0B6B4F), Gold (#C9A84C), Cream (#FBF8F1)
- **Style**: Cards with rounded corners, Islamic geometric touches

## Quick Start

```bash
npm install
node scripts/downloadQuran.js   # Downloads full Quran (Arabic + English + Urdu)
npm run dev                      # Start dev server
```

## Project Structure

```
src/
├── App.jsx                  # Main app with routing
├── main.jsx                 # Entry point
├── styles/theme.css         # Design tokens & global styles
├── components/
│   ├── BottomNav.jsx        # 5-tab navigation
│   ├── HomePage.jsx         # Prayer times + countdown + daily verse
│   ├── QuranReader.jsx      # Surah list + reading view
│   ├── Worship.jsx          # Tasbeeh counter + morning/evening adhkar
│   ├── Qibla.jsx            # Compass direction to Ka'bah
│   └── MorePage.jsx         # Settings
├── data/
│   ├── surahMeta.js         # All 114 surah metadata
│   ├── quranText.json       # Full Quran text (generated by script)
│   └── adhkar.js            # Morning/evening adhkar
├── utils/
│   ├── prayerCalc.js        # Solar position prayer time math
│   └── qiblaCalc.js         # Qibla bearing calculation
scripts/
└── downloadQuran.js         # Fetches Quran from alquran.cloud API
```

## Claude Code Prompts (in order)

### Phase 1: Get it running
```
cd muslimos && npm install && node scripts/downloadQuran.js && npm run dev
```
Verify all 5 tabs work, prayer times calculate, Quran surahs load.

### Phase 2: Fix the QuranReader import
The top-level await in QuranReader.jsx may need adjustment:
```
In QuranReader.jsx, change the SURAH_TEXT import to use a synchronous import 
or React lazy loading. The quranText.json file is large (~5MB) so consider 
lazy-loading each surah on demand from the JSON.
```

### Phase 3: Add audio playback
```
Add an audio player to the Quran reader. Use the Quran.com CDN for audio:
https://cdn.islamic.network/quran/audio-surah/128/ar.alafasy/{surahNumber}.mp3

Create a persistent AudioPlayer component that:
- Shows a mini player bar above the bottom nav
- Has play/pause, progress bar, next/prev surah
- Continues playing when switching tabs
- Lets user pick reciter (Mishary, Sudais, Abdul Basit)
- Stores last played position in localStorage

Keep the existing design language (emerald/gold/cream, Amiri + DM Sans).
```

### Phase 4: Add Urdu translation
```
The Urdu text is already downloaded in quranText.json (key: "u"). 
Add a language toggle pill in the QuranReader reading view:
- English (default)
- اردو (Urdu)
Display the selected translation below each Arabic verse.
Use Amiri font for Urdu text with direction: rtl.
```

### Phase 5: Add Hadith section
```
Add a Hadith tab to the More page or as a new bottom nav item.
Bundle a curated selection of ~100 hadith from:
- Sahih Bukhari
- Sahih Muslim
- Riyad-us-Saliheen

Each hadith needs: Arabic text, English translation, source reference.
Show as scrollable cards with search/filter by collection.
Use the same card design from the existing app.
```

### Phase 6: Deploy
```
npm run build
vercel --prod
```

## Audio CDN Reference

Per-surah MP3 (128kbps):
`https://cdn.islamic.network/quran/audio-surah/128/ar.alafasy/{1-114}.mp3`

Per-ayah MP3:
`https://cdn.islamic.network/quran/audio/128/ar.alafasy/{ayahNumber}.mp3`

Available reciters: ar.alafasy, ar.abdurrahmaansudais, ar.abdulbasitmurattal, ar.husary
